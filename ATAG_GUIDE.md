# ATAG 使用指南
## 从技术文档到动画网页的完整流程

---

## 快速开始（5分钟上手）

### 步骤 1：准备你的技术文档

将你的 AUTOSAR 技术文档整理成以下格式：

```markdown
# UDS 统一诊断服务

## 简介
UDS（Unified Diagnostic Services）是基于 ISO 14229 标准的车载诊断协议...

## 核心服务
- 0x10 诊断会话控制
- 0x27 安全访问
- 0x22 读取数据
...

## 通信流程
1. 诊断仪发送请求
2. ECU 处理请求
3. ECU 返回响应
...
```

### 步骤 2：选择合适的提示词模板

打开 `ATAG_PROMPTS.md`，找到对应主题的模板。例如 UDS 诊断：

```
生成 UDS 统一诊断服务的技术教学网页：

【内容结构】
第1页：封面...
第2页：协议简介...
...

【配色】电动蓝主题
【图标】Font Awesome
```

### 步骤 3：发送给 AI

将模板和你的文档内容一起发送给 AI（Claude、GPT-4 等）：

```
请根据以下模板和内容生成网页：

【模板】
[粘贴 ATAG_PROMPTS.md 中的模板]

【我的内容】
[粘贴你的技术文档]

【输出路径】
./output/uds-tutorial.html
```

### 步骤 4：获取并使用

AI 会生成完整的 HTML 文件，你可以：
1. 直接在浏览器中打开
2. 部署到内部服务器
3. 分享给团队成员
4. 用于培训演示

**需求：**

**步骤：**

1. **准备代码**
```c
// UDS 服务处理函数
uint8_t UDS_ProcessService(uint8_t* request, uint16_t reqLen,
                           uint8_t* response, uint16_t* respLen) {
    uint8_t sid = request[0];

    switch(sid) {
        case 0x10:  // 诊断会话控制
            return UDS_DiagnosticSessionControl(request, reqLen, response, respLen);
        case 0x22:  // 读取数据
            return UDS_ReadDataByIdentifier(request, reqLen, response, respLen);
        case 0x27:  // 安全访问
            return UDS_SecurityAccess(request, reqLen, response, respLen);
        default:
            return NRC_SERVICE_NOT_SUPPORTED;
    }
}
```

## 常见问题

### Q1：生成的代码太长，AI 输出不完整怎么办？

**方案 1：分页生成**
```
请先生成第 1-3 页
然后我会说"继续"，你再生成第 4-6 页
最后合并成完整文件
```

**方案 2：简化内容**
```
请生成简化版，每页内容减少 30%
重点突出核心概念
```

**方案 3：使用模板**
```
请基于 ATAG_METHODOLOGY.md 中的标准模板
只替换内容部分，不要重复生成样式代码
```

### Q2：如何确保风格统一？

**使用配置文件：**
```
创建 atag-config.json：
{
  "theme": "electric-blue",
  "font": "Microsoft YaHei",
  "iconLib": "Font Awesome",
  "animation": "fade-slide",
  "layout": "slideshow"
}

生成时引用：
请按照 atag-config.json 的配置生成网页
```

### Q3：如何添加公司特定元素？

**创建品牌模板：**
```
请在标准 ATAG 模板基础上添加：

【品牌元素】
- Logo：右上角，120x40px
- 配色：主色 #FF6B00，辅色 #00A8E8
- 字体：公司标准字体
- 页脚：版权信息 + 联系方式

【保存为】
company-template.html

【后续使用】
请基于 company-template.html 生成 XXX 主题网页
```

### Q4：如何导出为 PDF 或 PPT？

**方法 1：浏览器打印**
```
1. 在浏览器中打开网页
2. Ctrl+P 打开打印对话框
3. 选择"另存为 PDF"
4. 调整页面设置
5. 保存
```

**方法 2：使用工具**
```
使用 Puppeteer 或 Playwright：

const puppeteer = require('puppeteer');
const browser = await puppeteer.launch();
const page = await browser.newPage();
await page.goto('file:///path/to/uds-tutorial.html');
await page.pdf({path: 'uds-tutorial.pdf', format: 'A4'});
```

**方法 3：在线转换**
```
使用在线工具：
- HTML to PDF: https://www.html2pdf.com/
- CloudConvert: https://cloudconvert.com/html-to-pdf
```

## 最佳实践

### 1. 内容组织

✅ **好的做法：**
- 每页一个核心概念
- 先总后分，层次清晰
- 关键信息突出显示
- 配图辅助理解

❌ **避免：**
- 单页内容过多
- 文字密集无图示
- 逻辑跳跃
- 术语无解释

### 2. 视觉设计

✅ **好的做法：**
- 统一配色方案
- 合理使用留白
- 图标辅助识别
- 动画适度自然

❌ **避免：**
- 颜色过多杂乱
- 元素拥挤
- 过度动画
- 字体大小不一

### 3. 代码示例

✅ **好的做法：**
- 代码简洁易懂
- 添加详细注释
- 突出关键代码
- 提供完整上下文

❌ **避免：**
- 代码过长
- 无注释
- 语法错误
- 缺少说明

### 4. 交互体验

✅ **好的做法：**
- 翻页流畅
- 响应迅速
- 操作直观
- 提供反馈

❌ **避免：**
- 卡顿延迟
- 操作复杂
- 无提示信息
- 兼容性差
