🔧 UDS 10 服务
诊断会话控制 (Diagnostic Session Control)
控制 ECU 进入不同诊断会话模式，是所有 UDS 诊断的入口服务

📋 服务概述

🎯 服务 ID

0x10 - DiagnosticSessionControl

💡 核心功能

切换 ECU 的诊断会话模式，不同模式下可执行的诊断功能不同

🔑 常见会话类型

  • 0x01 - 默认会话 (Default Session)
  • 0x02 - 编程会话 (Programming Session)
  • 0x03 - 扩展诊断会话 (Extended Diagnostic Session)
  • 0x40-0x5F - 车辆制造商自定义会话

🔐 会话类型详解

🏠 默认会话

0x01 - Default Session

ECU 上电后的初始状态，功能受限

  • 读取基础故障码
  • 读取车辆信息
  • 不允许写入操作
  • 不允许刷写程序

🔧 扩展诊断会话

0x03 - Extended Diagnostic

用于深度诊断和参数调整

  • 读写 DID 数据
  • 执行例程测试
  • 清除故障码
  • IO 控制

💾 编程会话

0x02 - Programming Session

用于 ECU 软件刷写和标定

  • 下载应用程序
  • 刷写 Flash
  • 更新标定数据
  • 需要安全访问

📦 报文格式

📤 请求报文 (Request)

字节位置参数名称说明
Byte 0SID服务标识符 = 0x10
Byte 1Sub-function会话类型 (0x01/0x02/0x03/0x40-0x5F)

📥 响应报文 (Positive Response)

字节位置参数名称说明
Byte 0SID + 0x40肯定响应 = 0x50
Byte 1Sub-function回显请求的会话类型
Byte 2-3P2 Server Max默认超时时间 (ms, 高字节在前)
Byte 4-5P2* Server Max增强超时时间 (ms × 10)

🔄 通信流程

诊断仪
(Tester)
→
ECU
→
诊断仪
(Tester)

📝 交互步骤

  1. 诊断仪发送 0x10 + 会话类型 请求
  2. ECU 验证请求合法性（会话类型是否支持）
  3. ECU 切换到目标会话模式
  4. ECU 返回 0x50 + 会话类型 + P2/P2* 超时参数
  5. 诊断仪根据响应调整后续通信超时时间

💻 实例 1：切换到扩展诊断会话

📤 请求报文

10 03
// 10 = 服务ID (DiagnosticSessionControl)
// 03 = 扩展诊断会话

📥 响应报文

50 03 00 32 01 F4
// 50 = 肯定响应 (0x10 + 0x40)
// 03 = 回显会话类型
// 00 32 = P2 Server Max = 50ms
// 01 F4 = P2* Server Max = 5000ms (500 × 10)

💻 实例 2：切换到编程会话

📤 请求报文

10 02
// 10 = 服务ID (DiagnosticSessionControl)
// 02 = 编程会话

📥 响应报文

50 02 00 64 03 E8
// 50 = 肯定响应
// 02 = 回显会话类型
// 00 64 = P2 Server Max = 100ms
// 03 E8 = P2* Server Max = 10000ms (1000 × 10)

❌ 实例 3：否定响应

📤 请求报文

10 05
// 10 = 服务ID
// 05 = 不支持的会话类型

📥 否定响应报文

7F 10 12
// 7F = 否定响应标识
// 10 = 请求的服务ID
// 12 = NRC (Sub-Function Not Supported)

常见否定响应码 (NRC)：
0x12 - 子功能不支持
0x13 - 报文长度或格式错误
0x22 - 当前会话不允许此操作
0x33 - 安全访问被拒绝

🎯 应用场景

🔍 故障诊断

切换到扩展诊断会话，读取详细故障信息

  • 读取冻结帧数据
  • 执行诊断例程
  • 清除故障码

💾 ECU 刷写

切换到编程会话，进行软件更新

  • 下载应用程序
  • 刷写 Flash 存储器
  • 更新标定数据

⚙️ 参数配置

在扩展会话中调整 ECU 参数

  • 写入配置数据
  • 执行 IO 控制
  • 调整运行参数
