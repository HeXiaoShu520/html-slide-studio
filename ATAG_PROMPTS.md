```

【内容结构】

- 副标题：基于 IP 的车载诊断

第2页：为什么需要 DoIP
- 传统 CAN 诊断的限制（速率低、刷写慢）

- DoIP 优势：高速（100Mbps+）、大容量

第3页：协议栈
分层展示：
- 应用层：UDS 诊断服务
- DoIP 层：路由激活、诊断消息封装

- 网络层：IPv4 / IPv6
- 数据链路层：Ethernet

第4页：连接建立流程
使用时间线组件：
1. 车辆公告（Vehicle Announcement）- UDP 广播
2. 路由激活请求（Routing Activation Request）
3. 路由激活响应（Routing Activation Response）
4. 诊断通信开始

第5页：报文格式
DoIP Header：
- Protocol Version（1字节）
- Inverse Protocol Version（1字节）
- Payload Type（2字节）
- Payload Length（4字节）
- Payload（变长）

常用 Payload Type：
- 0x0001：车辆识别请求
- 0x0004：车辆识别响应
- 0x0005：路由激活请求
- 0x8001：诊断消息（UDS）

第6页：诊断消息封装
展示 UDS over DoIP：
```
DoIP Header:
  Protocol Version: 0x02
  Payload Type: 0x8001
  Payload Length: 0x00000005

DoIP Payload:
  Source Address: 0x0E80 (诊断仪)
  Target Address: 0x1000 (ECU)
  UDS Message: 22 F1 90 (读取VIN)
```

第7页：实际应用
- EOL 生产线：快速刷写程序
- 售后维修：远程诊断支持
- OTA 升级：整车软件更新
- 开发调试：XCP 标定

【配色】电动蓝主题
【动画】网络连接建立动画、数据传输效果
```
